# MongoDB

# 1.mq如何保证消息不丢失

将消息进行持久化，有同步刷盘(失败可以走补偿机制，缺点是会有延迟)和异步刷盘 消息确认机制 可以在发送消息之前记录日志，如果异常失败后期可以通过日志做补偿 (rabbitmq消息发送成功后会清除，kafka不会清除，是通过定时日志清除策略来清除)
